# 街づくりシミュレーションゲーム Phase 1 仕様書

## コンセプト
グリッド型の2D都市シミュレーション。ゾーニングで街を育てる基本ループに加え、
**鉄道高架と高架下開発**を核となる差別化要素とする。
高架を通すと高架下スペースが生まれ、店舗や駐車場を設置して賃料収入を得られる
（有楽町・秋葉原の高架下商業のイメージ）。

## 技術要件
- HTML + JavaScript（単一フォルダ構成、index.htmlを開けば動作）
- 描画：**Three.js による見下ろし3D**（CDN読み込み）
  - カメラ：斜め俯瞰（アイソメトリック風）。マウスドラッグで回転、ホイールでズーム、右ドラッグでパン
  - 建物はボックス主体のローポリ表現。建物レベルに応じて高さが変わる
  - 高架はY軸で地上より1段高い位置に描画し、橋脚を柱で表現
  - 高架下店舗・駐車場は高架の真下に配置され、カメラを回すと高架の下に見える
  - ライティング：環境光＋平行光1灯のシンプル構成。影は任意（重ければ省略可）
- セーブ：localStorageに自動保存（手動セーブ/ロードボタンも設置）
- 外部通信なし。ローカルで完結

## マップ
- 32×32のグリッド。1タイル＝1区画
- 初期状態：全タイル空地。マップ端に「外部接続道路」1本が既設

## プレイヤーの操作（ツールパレット）
1. 道路を敷く（1タイル・ドラッグで連続敷設）
2. ゾーン指定：住宅（緑）／商業（青）／工業（黄）
3. **鉄道高架を敷く**（後述）
4. **高架下施設を設置**（後述）
5. 撤去（ブルドーザー）
6. ポーズ／速度変更（1x / 2x / 3x）

## 基本ルール
- ゾーンは道路に隣接したタイルのみ機能する
- 需要バー（住宅・商業・工業の3本）に応じて、ゾーン指定済みタイルに建物が自動で建つ
- 建物にはレベル1〜3があり、地価が上がると成長する

## 鉄道高架システム（本作の核）
- 高架は**道路や建物の上を通過できる**（立体交差）。空地の上も可
- 高架の敷設コストは道路の5倍
- 高架は「駅」を設置でき、駅の周囲5タイルの地価と商業需要が上昇する
- 高架がマップ端まで接続されていると「外部接続」となり、人口流入ボーナス

### 高架下スペース
- 高架が通ったタイルは自動的に「高架下スペース」となる
- 高架下スペースには以下を設置できる：
  - **高架下店舗**：隣接道路が必要。賃料収入（毎月）。駅から3タイル以内なら収入2倍
  - **高架下駐車場**：道路隣接が必要。収入は店舗の半分だが、周囲の商業建物のレベル上限を+1する
  - 未利用のまま放置も可（収入ゼロ、周辺地価に微マイナス）
- 高架下店舗は周辺の地価も上昇させる（暗い高架下→賑わいのある空間への転換を表現）

## 経済
- 資金：初期 ¥50,000
- 収入（毎月）：税収（人口×住宅税 + 商業・工業の売上税）+ 高架下賃料
- 支出（毎月）：道路・高架の維持費
- 資金がマイナスになったらゲームオーバー（Phase 1では警告表示のみでも可）

## 地価の計算（シンプル版）
各タイルの地価 = 基準値
+ 駅からの距離ボーナス（近いほど高い）
+ 隣接する高架下店舗ボーナス
− 工業地帯への近さペナルティ
− 未利用高架下への隣接ペナルティ

## UI
- 画面上部：資金、人口、日付、需要バー3本
- 画面左：ツールパレット
- タイルクリックで情報表示（地価、収入、建物レベル）
- 地価ヒートマップ表示の切り替えボタン

## Phase 1のゴール（完成の定義）
- 道路とゾーンで人口1,000人の街が作れる
- 高架を通して駅を作り、高架下店舗で「税収より賃料が儲かる」体験ができる
- リロードしても街が保存されている

## Phase 2以降の予告（実装しないが設計時に考慮）
- 高架下店舗の種類分化（飲食・物販・オフィス）
- 電車の走行アニメーションと乗客数シミュレーション
- 建物モデルの多様化（屋根形状、看板、街路樹など）
- 昼夜サイクルと高架下店舗の夜間ライトアップ
- **PLATEAU連携（実在都市モード）**：国交省PLATEAUのLOD1データ（建物フットプリント＋高さ）をグリッドに割り当て、有楽町など実在の街を初期マップとして読み込む。ゲームロジックはグリッドのまま流用する
